Keyword Research, On-Page SEO Strategy, and Implementation for My Technical SEO Portfolio

This case study shows how I combined keyword research, search intent analysis, page-level mapping, on-page implementation, and structured-data cleanup to make my technical SEO portfolio clearer, stronger, and easier to evaluate.

Overview

I optimized my personal technical SEO portfolio by combining keyword research, search intent analysis, page-level mapping, on-page implementation, and structured-data cleanup.

The objective was to make the site easier to understand for recruiters, hiring managers, and potential clients while improving how each page aligned with a distinct keyword theme.

Project Goal

The portfolio already communicated a technical SEO focus, but its keyword targeting had not yet been fully validated. I wanted to:

  • identify the strongest homepage keyword
  • assign distinct supporting keywords to key pages
  • implement those changes without redesigning the site
  • improve metadata, internal linking, and structured data consistency

Starting Point

The site already had a strong structure, including a homepage, About page, Projects page, tools, guides, and multiple audit-focused case studies.

What it lacked was a more deliberate keyword hierarchy. The homepage, supporting pages, and project pages needed clearer separation of search intent so the site felt more strategic and easier to navigate.

Audience

This portfolio was designed for:

  • recruiters and hiring managers
  • SEO leads and marketing decision-makers
  • potential freelance clients looking for technical SEO support

That audience needed three things quickly:

  • clear specialist positioning
  • visible proof of work
  • easy navigation into relevant case studies

Research Approach

I started with seed keywords related to:

  • SEO portfolio intent
  • technical SEO
  • audits
  • case studies
  • site architecture

I used Ahrefs Free Keyword Generator to test broad and niche variations, then used Google Trends and live SERP reviews to validate demand and search intent.

This helped me avoid forcing pages into keywords that were weak, overly commercial, or misaligned with the actual page purpose.

Trend and Search Intent Validation

Why seo portfolio won

A Google Trends comparison showed stronger relative interest for seo portfolio than weaker alternatives like seo specialist portfolio.

What the SERP showed

The search results for seo portfolio leaned toward:

  • showcase pages
  • portfolio examples
  • visible project work
  • inspiration and proof-of-work intent

That supported using the homepage as a broad SEO portfolio hub and the Projects page as a clearer examples page.

I also reviewed the SERP for keyword research case study, which showed more example-driven and educational intent. That helped shape the structure of case-study pages as walkthroughs rather than simple summaries.

Keyword Validation

I organized the findings in a validation sheet and classified each term by:

  • category
  • priority
  • decision status
  • rationale

This made it easier to separate:

  • strong portfolio-intent terms
  • strong supporting audit terms
  • useful niche supporting keywords
  • terms that should be dropped

Final Keyword Strategy

Primary keyword

  • seo portfolio

Secondary keywords

  • seo portfolio examples
  • seo portfolio website
  • technical seo
  • technical seo audit
  • keyword research case study
  • seo audit
  • website seo audit

Supporting keywords

  • site architecture audit
  • technical seo specialist
  • keyword research case study example
  • seo audit checklist

Terms I did not prioritize

I dropped or deprioritized terms that were too weak, too commercial, or too misleading for a personal portfolio.

Examples included technical seo portfolio, website audit case study, and service-heavy phrases like seo audit services.

Implementation Plan

I translated the keyword map into direct on-page SEO updates across the most important pages.

Homepage

  • revised H1
  • refined hero copy
  • improved CTA language
  • stronger links into major case studies

About page

  • strengthened specialist positioning
  • updated H1 and intro
  • improved strengths and workflow section labels

Projects page

  • reframed the page as SEO portfolio examples
  • improved intro copy
  • clarified project card descriptions

Case study pages

  • updated titles, H1s, and intros
  • refined section labels
  • improved internal links
  • aligned metadata and structured data

Actual Implementation

I implemented the approved updates directly in the codebase while preserving the existing layout and visual design.

The work included:

  • title tags
  • meta descriptions
  • H1s
  • intro copy
  • section headings
  • project card descriptions
  • internal links
  • JSON-LD cleanup

Before vs After

Homepage

Before: The homepage had strong technical SEO positioning, but the headline and intro were not fully aligned with the final homepage keyword strategy.

After: The homepage was updated to target seo portfolio more directly with a stronger H1, clearer supporting copy, and a CTA that reinforced portfolio-example intent.

About Page

Before: The About page communicated technical ability, but the wording was less clearly framed around specialist positioning.

After: The page was refined around technical seo and technical seo specialist with a stronger H1, tighter intro copy, and clearer section framing.

Projects Page

Before: The Projects page showed solid work samples, but it did not explicitly position itself as a collection of SEO portfolio examples.

After: The page was reframed as a portfolio examples hub, with stronger introductory language and clearer project descriptions grouped by SEO problem type.

QA and Technical Cleanup

After implementation, I ran a QA pass to check:

  • one H1 per page
  • page-level keyword separation
  • internal links
  • metadata consistency
  • structured-data alignment

What passed

  • one H1 per page
  • clean keyword targeting by page
  • stronger internal links between hub and case-study pages
  • natural, non-spammy copy
  • good alignment between title tags, meta descriptions, OG tags, and Twitter tags

Final issue found

Some pages had updated visible metadata, but their JSON-LD still reflected older wording.

Fix applied

I updated schema name, description, and jobTitle fields so the structured data matched the revised page intent.

This included aligning retargeted case-study schema and updating the About page jobTitle to Technical SEO Specialist.

Box & Bloom Correction

One page required a manual correction during QA.

The Box & Bloom project had briefly been pushed too far toward a keyword research framing. After review, I corrected that because the full scope of the work was broader and better described as an e-commerce SEO case study.

I updated:

  • title
  • meta description
  • H1
  • section labels
  • supporting copy
  • JSON-LD
  • project card label on the Projects page

This was an important correction because it kept the page honest to the real project scope instead of forcing an exact-match keyword label onto the work.

Outcome

This project improved the portfolio in four major ways:

  • clearer homepage positioning around seo portfolio
  • better separation between portfolio, specialist, and case-study intent
  • stronger project framing and internal linking
  • cleaner alignment between visible metadata and structured data

Even without ranking or traffic data yet, the site is now more strategically organized and easier to evaluate for both users and search engines.

Key Takeaways

1. The homepage should target the broadest intent

seo portfolio was the strongest fit for the homepage because it matched both search demand and the actual purpose of the site.

2. Supporting pages need narrower intent

Case studies performed better when mapped to more specific audit and architecture terms rather than repeating the homepage theme.

3. Search volume alone is not enough

Some phrases looked plausible but were too weak, too commercial, or too inaccurate for the real page purpose.

4. Implementation matters as much as research

Keyword research only became useful once it changed the titles, headings, copy, links, and structured data.

5. Accuracy is part of good SEO

The Box & Bloom correction reinforced that keyword strategy should support truthful positioning, not override it.

Final Reflection

This project was an end-to-end SEO workflow applied to my own portfolio:

  • keyword research
  • trend and SERP validation
  • keyword mapping
  • on-page implementation
  • QA review
  • structured-data cleanup

The final result is a portfolio that more clearly communicates what kind of SEO work I do, where to find proof of that work, and how each major page supports a distinct intent.

That makes the site stronger not only as an SEO asset, but also as a portfolio for recruiters, hiring managers, and potential clients.
