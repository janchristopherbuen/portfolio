SEO Blog Optimization

Canonical tags, robots directives, and structured data improvements

Project Overview

This project involved optimizing a technical blog for search engines by addressing canonicalization issues, robots.txt rules, and structured data markup.

Technical SEO Problem

The blog had duplicate content due to missing canonical tags, inconsistent robots directives, and lacked schema markup for articles.

Audit Methodology

  • Run crawl using Screaming Frog
  • Inspect headers and meta tags
  • Validate schema with online tools

Technical Findings

Numerous pages had no canonical URL. Robots.txt allowed all bots but lacked sitemap entry. JSON-LD markup was absent.

[screenshot placeholder]

Fixes Implemented

  • Added <link rel="canonical"> to head of each article
  • Updated robots.txt with explicit sitemap location
  • Inserted Article JSON-LD markup via templating

Results and Impact

Within three weeks, duplicate pages were de-indexed and organic traffic to blog posts increased by 18%.

Tools Used

Google Search Console, Screaming Frog, Schema.org validator.
