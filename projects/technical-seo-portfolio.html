GitHub Case Study

Technical SEO Portfolio

Assessment of technical factors that influence search engine crawling and site performance. This includes checking HTTP status codes, crawlability, and identifying technical barriers that could prevent pages from appearing in search results.

500 URLs crawled Performance: 99 Core Web Vitals: Good

Project Overview

Repository link
https://github.com/janchristopherbuen/technical-seo-portfolio
Project goal
Analysis of the website's structure to evaluate page hierarchy, internal linking strategy, and crawl depth. The objective is to ensure that important pages are easily discoverable by search engine crawlers and that link equity flows efficiently across the site.
Technologies used
Google Lighthouse, Chrome DevTools, XML Sitemap Validator, Robots.txt Tester, Screaming Frog SEO Spider, Google PageSpeed Insights, Google Chrome Developer Tools, Chrome Network Panel, Google Sheets

Technical Findings

  • The website does not provide a robots.txt file.
  • The website does not provide an XML sitemap.
  • Pages lack canonical tags, increasing duplicate content risks.
  • All discovered pages returned indexable responses and were accessible to crawlers.
  • The crawl reached the 500-URL limit of the Screaming Frog free version.
  • Several pages contain duplicate or overly long titles.
  • Approximately 22% of pages lack meta descriptions.
  • The site passes Core Web Vitals.

SEO Work Performed

Technical SEO fixes

  • Create a robots.txt file to manage crawler access.
  • Create an XML sitemap.
  • Add a self-referencing canonical tag.
  • Run a full crawl without URL limits to evaluate the entire site structure and internal linking architecture.

On-page optimization

  • keeping titles under 60 characters
  • ensuring each page has a unique title
  • including relevant keywords
  • maintaining 150–160 character descriptions

Crawl analysis

  • Total URLs encountered: 500
  • Total URLs crawled: 500
  • Internal URLs: 499
  • URLs blocked by robots.txt: 0
  • All crawled pages returned HTTP 200 success responses.

Internal linking improvements

  • Category pages function as internal linking hubs.
  • The majority of pages exist at crawl depth level 2.
  • Strengthening internal links to high-value pages can further improve discoverability.
  • Ensure important pages remain within three clicks from the homepage.
